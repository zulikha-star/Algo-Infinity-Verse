📐 Common Complexity Patterns

O(1)

Constant

arr[i] = 5;
x = a + b;
map.get(key);

Single operations, array access, hash map lookup. Never loops.

O(log n)

Logarithmic

while (lo <= hi) {
  mid = (lo+hi)/2
  lo = mid + 1
}

Loop that halves the search space each iteration.

O(n)

Linear

for (i = 0; i < n; i++) {
  // O(1) work
}

Single loop over input. Each iteration is O(1).

O(n²)

Quadratic

for (i=0; i<n; i++)
  for (j=0; j<n; j++)
    // O(1) work

Nested loops both over n. Classic Bubble Sort pattern.

O(2ⁿ)

Exponential

function f(n) {
  return f(n-1)
       + f(n-2);
}

Branching recursion with 2 calls per level. Fibonacci without memoization.

O(n log n)

Linearithmic

// Divide array (log n)
// Merge each half (n)
// → Merge Sort

Divide and conquer — split O(log n) times, O(n) work per level.
