Introduction to Computational Geometry

Computational Geometry is a branch of Computer Science that studies algorithms for solving geometric problems involving points, vectors, lines, polygons and planes.

It forms the backbone of modern Computer Graphics, Robotics, CAD software, GIS systems, Navigation, Collision Detection, Game Development, and many Competitive Programming problems.

Computer Graphics

Rendering 2D and 3D scenes efficiently.

Robotics

Obstacle detection and motion planning.

GIS

Processing maps and geographical data.

Game Development

Collision detection and visibility testing.

CAD

Engineering and architectural design software.

Competitive Programming

Many advanced geometry problems rely on these algorithms.

Dot Product

The Dot Product measures how much two vectors point in the same direction. It is commonly used to calculate angles, projections and similarity between vectors.

Formula

A · B = Ax × Bx + Ay × By

C++ Example


struct Point{
    int x,y;
};

int dot(Point a, Point b){
    return a.x*b.x + a.y*b.y;
}
Time Complexity: O(1)
  • Angle between vectors
  • Lighting in Computer Graphics
  • Projection calculations
  • Physics engines

Cross Product

Cross Product tells whether one vector lies to the left or right of another. It is one of the most important operations in Computational Geometry.

Formula

Cross(A,B)=Ax×By−Ay×Bx

C++ Example


struct Point{
    int x,y;
};

int cross(Point a,Point b){
    return a.x*b.y-a.y*b.x;
}
Time Complexity: O(1)

Segment Intersection

Two line segments intersect if their orientations satisfy specific geometric conditions.

Algorithm

  • Find four orientations.
  • Check general case.
  • Handle collinear case.

bool intersect(Point p1,Point q1,
Point p2,Point q2){

int o1=orientation(p1,q1,p2);

int o2=orientation(p1,q1,q2);

int o3=orientation(p2,q2,p1);

int o4=orientation(p2,q2,q1);

return (o1!=o2 && o3!=o4);

}
Time Complexity : O(1)

Convex Hull

The Convex Hull is the smallest convex polygon that contains all the given points. Imagine stretching a rubber band around a set of nails—the shape formed is the convex hull.

Popular Algorithms

  • Graham Scan — O(n log n)
  • Jarvis March — O(nh)
  • Monotonic Chain — O(n log n)

Key Idea

Sort the points and repeatedly use the cross product to remove points that create clockwise turns.

C++ Skeleton


sort(points.begin(), points.end());

for(Point p : points){

while(hull.size()>=2 &&
cross(hull[hull.size()-2],
      hull.back(),
      p)<=0){

    hull.pop_back();

}

hull.push_back(p);

}
Time Complexity : O(n log n)

Closest Pair of Points

Find the two points having the minimum Euclidean distance. A brute-force solution takes O(n²), while Divide & Conquer solves it in O(n log n).

Approach

  1. Sort points by x-coordinate.
  2. Split into two halves.
  3. Solve recursively.
  4. Merge using strip.
Time Complexity : O(n log n)

Sweep Line Algorithm

Sweep Line processes geometric objects by moving an imaginary vertical line from left to right while maintaining active events.

Applications

  • Segment Intersection
  • Rectangle Union
  • Closest Pair
  • Event Processing
Typical Complexity : O(n log n)

Rotating Calipers

Rotating Calipers is a geometric technique applied after computing the Convex Hull. It efficiently finds

  • Diameter of polygon
  • Minimum Width
  • Maximum Distance
  • Bounding Rectangle
Time Complexity : O(n) (after Convex Hull)

Quick Quiz

1. Which operation determines clockwise and counter-clockwise?

The sign of the Cross Product between two vectors tells you whether the turn is clockwise, counter-clockwise, or collinear.

2. Which algorithm computes Convex Hull in O(n log n)?

Graham Scan sorts points and builds the hull in O(n log n), dominated by the sorting step.

3. Polygon Area is calculated using?

The Shoelace Formula sums cross-product-like terms across polygon vertices to compute the enclosed area.

4. Closest Pair optimal complexity?

Divide & Conquer solves Closest Pair in O(n log n), compared to O(n²) for brute force.

5. Sweep Line mainly processes?

Sweep Line algorithms move a line across the plane, processing start/end "events" in sorted order.
