📖 Why One Size Does Not Fit All

♻️

LRU — The Default Everyone Uses

Linked list + hash map. O(1) hit, O(1) eviction. Perfect for temporal locality — recently accessed items are likely to be accessed again. Fatal flaw: a sequential scan of N items evicts all N cache residents, even items accessed millions of times. MySQL calls this "cache pollution" and uses a midpoint-insertion variant to prevent it.

📊

LFU — Frequency Over Recency

Tracks exact access count per item. Items accessed most often survive longest. Wins on frequency-skewed workloads (80% of accesses hit 20% of items). Fatal flaw: frequency counts never decay — an item accessed 1000 times hours ago beats a new hot item accessed 5 times. "Cache sedimentation" traps stale-but-historically-popular items.

🎯

ARC — IBM's Patent, ZFS's Secret

Maintains 4 lists: T1 (seen once recently), T2 (seen twice+), B1 (ghost of evicted T1), B2 (ghost of evicted T2). A ghost hit on B1 means "we should have kept that" → shift budget to T1. Ghost hit on B2 → shift to T2. The split target t adapts in real time. Used in ZFS, NetApp ONTAP, and IBM's storage systems.

🕐

CLOCK — O(1) LRU Approximation

A circular buffer of cache slots, each with a single reference bit. On access: set bit to 1. On eviction: sweep clockwise — if bit is 1, clear it and continue (second chance); if bit is 0, evict. Approximates LRU without the overhead of moving list nodes. Used in Linux's page cache, PostgreSQL's buffer manager, and most OS virtual memory systems.
